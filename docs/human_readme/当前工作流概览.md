# 当前工作流概览

本项目当前采用面向 AFSIM 的专用 Agent 流水线，而不是直接由自然语言端到端生成脚本。

## 主流程

```text
自然语言
→ Intent Parsing
→ AFSIM-IR
→ Grounding
→ Grounded IR
→ Hierarchical Generation
→ Static Verification
→ Self Repair
→ mission.exe
→ Execution Repair
→ Executable Scenario
```

## 当前原则

- 生成必须经过 `IR + Grounding`，禁止绕过中间表示直接写脚本
- `mission.exe` 是最终裁判，静态检查只是前置筛查
- 规则来源优先级：
  `官方文档 > 官方 Demo > Grounding 库 > 历史实验 > 模型推断`
- 修复应尽量沉淀为通用规则，而不是针对单个 benchmark 特判

## 当前重点

当前工作重点不再是“把流程跑通”，而是继续缩小三类差距：

1. `Static Checker` 与 `mission.exe` 的差距
2. Grounding / Generation 的实现约束缺口
3. Repair 过程中的漂移与结构性修坏
